# 1. Apresentação

O **Atlas Distribuidora Analytics** é um projeto de portfólio desenvolvido para simular a construção de uma solução completa de dados dentro de uma empresa de distribuição de materiais para construção civil.

Diferentemente de projetos focados apenas em dashboards ou consultas SQL, este projeto reproduz todas as etapas do ciclo de vida dos dados, desde o entendimento do negócio até a disponibilização de informações para tomada de decisão.

O objetivo é desenvolver uma solução integrada utilizando tecnologias amplamente empregadas no mercado, como MySQL, SQL, Python, Power BI e GitHub, documentando cada etapa do processo de forma semelhante ao que ocorre em empresas reais.

---

# 2. Sobre a Atlas Distribuidora

A Atlas Distribuidora é uma empresa fictícia especializada na distribuição atacadista de materiais para construção.

Seu portfólio contempla produtos destinados aos segmentos de construção, reforma, acabamento e manutenção predial.

Entre as principais categorias comercializadas estão:

- Tintas
- Pisos e revestimentos
- Louças sanitárias
- Metais
- Materiais hidráulicos
- Materiais elétricos
- Ferragens
- Ferramentas
- Impermeabilizantes
- Argamassas
- Rejuntes

A empresa atende depósitos, home centers, lojas especializadas e construtoras de pequeno e médio porte.

Sua operação é baseada em representantes comerciais distribuídos entre diferentes estados da região Nordeste.

---

# 3. Contexto do Negócio

Nos últimos anos a Atlas apresentou crescimento constante em seu volume de vendas.

Com o aumento do número de clientes, produtos, representantes e filiais, também aumentou a quantidade de informações geradas diariamente.

Grande parte dessas informações encontra-se distribuída entre planilhas, sistemas operacionais e controles manuais.

Como consequência, diversos processos passaram a depender de consolidações manuais, dificultando a geração de indicadores confiáveis e reduzindo a velocidade das análises.

---

# 4. Problema

A diretoria identificou a necessidade de centralizar os dados operacionais da empresa e disponibilizar indicadores confiáveis para apoio à tomada de decisão.

Atualmente existem dificuldades para responder perguntas como:

- Qual é o faturamento consolidado da empresa?
- Quais representantes estão atingindo suas metas?
- Quais clientes concentram maior parte da receita?
- Quais categorias apresentam maior crescimento?
- Quais produtos possuem baixo giro?
- Qual filial apresenta melhor desempenho?
- Como evoluíram as vendas ao longo do tempo?

A inexistência de uma plataforma analítica integrada aumenta o tempo necessário para geração de relatórios e reduz a capacidade de resposta da gestão.

---

# 5. Objetivo do Projeto

Construir uma plataforma completa de dados capaz de integrar informações comerciais, operacionais e estratégicas da Atlas Distribuidora.

O projeto contemplará todas as etapas do ciclo analítico:

- Levantamento de requisitos
- Modelagem de banco de dados
- Desenvolvimento de consultas SQL
- Geração de dados fictícios
- Modelagem dimensional
- Construção de dashboards
- Análises exploratórias em Python
- Documentação técnica
- Publicação em GitHub

---

# 6. Objetivos Específicos

Ao término do projeto será possível:

- armazenar dados em um banco relacional;
- executar consultas SQL para responder perguntas de negócio;
- analisar indicadores estratégicos;
- acompanhar metas comerciais;
- monitorar desempenho por região;
- avaliar desempenho de representantes;
- identificar oportunidades de crescimento;
- visualizar informações em dashboards interativos.

# 10. Considerações

Todo o conteúdo desenvolvido neste projeto possui finalidade exclusivamente educacional.

Nenhum dado utilizado representa informações reais de empresas, clientes ou fornecedores.

A proposta é reproduzir cenários encontrados em organizações reais, permitindo demonstrar competências em Engenharia de Dados, Banco de Dados, Business Intelligence e Análise de Dados utilizando boas práticas de documentação e desenvolvimento.
